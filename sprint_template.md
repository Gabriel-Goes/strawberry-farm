# Sprint Template

## Sprint Goal
Clear description of what the sprint is trying to achieve.

---

## In Scope

List of features or changes included.

---

## Out of Scope

Explicitly list things that should NOT be changed.

---

## Implementation Plan

Technical approach to achieve the goal.

---

## Risks

Potential technical or design risks.

---

## QA Focus

Areas that require special testing attention.

---

# Implementation Notes

Summary of actual code changes.

---

# QA Report

Results of tests performed.

- gameplay
- persistence
- UI
- regressions

---

# Sprint Review

Evaluation of the sprint outcome.

- what worked
- what did not work
- remaining risks
- next priorities
